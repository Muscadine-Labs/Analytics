# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

If you discover a security vulnerability in Curator, please report it privately:

1. **Email** muscadinelabs@gmail.com with details, or

3. **Include the following in your report:**
   - Type of issue (e.g., XSS, injection, DoS)
   - Affected paths or components
   - Location (file paths, routes, or commit references)
   - Step-by-step reproduction instructions
   - Proof-of-concept or exploit code (if available)
   - Impact assessment

4. **Response timeline**
   - We aim to acknowledge reports within 5 business days
   - We will provide updates on remediation progress
   - We may follow up with clarifying questions

5. **Safe harbor**
   - We support safe harbor for security researchers who make good-faith efforts to follow these guidelines and avoid privacy violations, destruction of data, or disruption of services.
